Tema:  Arquitectura de Monolito

Aprendido:  
El monolito es un patron de diseno en donde todo el codigo base, se guarda y se ejecuta en un solo repositorio. Osea que toda la aplicacion funciona con el repo en un solo lugar. Esto benefia el desarrollo porque esto permite al sistema aceptar peticiones de un solo sitio y formato. El sistema no tiene que soportar la carga de procesar datos de diferentes servicios.
- Los componentes de un monolito son:
	- **Cliente(UI)**: Donde la informacion se sirve en el lado del usuario
	- **Database**: Un Monolito usa,por lo general, Relation database management system. Osea una base de datos relacional.
	- Server: Donde voy a tener toda la logica y funcionalidad del sistema.
Ventajas:
- Facil de desplegar
- Simple de desplegar
- Facil de debuggear y testear
- Mas seguro, al ser un sistema cerrado sus actividades estan contenidas
Desventejas:
- Dificial de adaptar nuevas tecnologias. La mayoria de veces hay que refactorizar todo para incorporar nuevas tecnologias.
- Tedioso de escalar. A veces para agregar una simple feat hay que reconstruir otras cosas y se lleva mucho tiempo.


Derivado:  Microservicios y Arquitectura monolotica modular
